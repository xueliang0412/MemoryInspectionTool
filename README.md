需求背景：

针对之前软件使用时内存显著增高，由内存泄漏导致软件Crash等问题，急需一个软件内存监控工具进行软件发版前的性能测试，对在跑软件内存进行实时监控，并生成统计结果，统一保存至内存监控报告中。



需求功能：

1. 可在UI界面选择1或多个软件进行监控；

2. UI界面实时生成监控软件的内存折线图，显示各监控软件的内存走势；

3.  自动生成软件内存监控报告；

4. 监控软件内存数据存入软件内存监控报告（.xlsx文件）；

5. 可界面选择配置1或多个监控软件生成对比折线图，保存至监控报告；

6. 生成各自监控软件进程的折线图，便于宏观细致感受内存走势；

7. 监控报告可自行选择保存路径；
